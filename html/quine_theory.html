The Quine-McCluskey Technique

What is The Quine-McCluskey Technique?


The Quine-McCluskey technique developed Edward J. McCluskey and Willard V. Quine is yet another method of simplifying Boolean functions. However, this method is better preferred than the Karnaugh Map Method as it is much easier to simplify Boolean functions which have more than 5 variables.

This method utilizes a tabular method using prime implicants, where a prime implicant is a product or sum term which cannot be reduced any further even by combining any other product or sum terms from the Boolean function.

Next you set out a table with 6 rows with the first column being the letter configurations, 2nd the index or the number represented by the letter configuration and the other 4 are the separate letters.

Also, you will group the numbers based on how many 1s are contained in it. For example, the red group is the number with only 1 one, the yellow represents numbers with 2 ones, blue represents numbers with 3 ones and green represents numbers with 4 ones.

Next you will compare numbers in group Red or group 1 with those in Group Yellow or Group 2. You will look for the numbers where there is only one bit different when comparing. That different bit would be omitted and the remaining would be the result.

For example, with W'X'YZ' (0010) in the Red Group that would compare with W'XYZ'- 0110 in the 2nd group and as they have only one bit different which is the X and X' that would be omitted, and it would leave W'YZ'.

Here you would do the same as you did for the 1st table where you compare elements in the 1st group or Red group with those in the Yellow group or 2nd group, then the Yellow group with the Blue Group.

For example, In the first group there is W'YZ' and the second group there is WYZ' therefore there is 1 bit/letter different, which would be the W, thus you omit that W and result in YZ'.

So, based on the matches for the 2nd table so far, we have YZ' + WX' + WY

So, for the third table, where ever you see any of the elements in the first column in any of the letter configurations in the first row you would place an X.

For example, for YZ’ you can find that in WXYZ’, WX’YZ’, W’XYZ’ and W’X’YZ’.

An X - would represent the first found match.

A □ – would represent if you find a match in a column which already has an X, you would use the □ instead of putting another X in the column.

A ● – would represent if there is no other matches found in the column but one, then put a ●.
